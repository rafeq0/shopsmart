# Sprint Retrospective - ShopSmart

## Went Well
- User stories were delivered with isolated feature branches and merged through `develop`.
- Frontend scope remained simple and aligned with sprint constraints (HTML/CSS/JS only).
- Core shopping flow (registration, catalog, cart) is functional and demo-ready.

## To Improve
- Repository automation permissions were insufficient for full API-driven PR workflow.
- Story-key consistency between Jira naming and commit naming needs earlier alignment.

## Action Items
- Configure GitHub token/app scopes to allow PR creation, review comments, and approvals.
- Add a sprint kickoff checklist to validate Jira keys before development starts.
- Keep one "sprint closure" document updated during the sprint, not only at the end.
